# SS_Pharmacy - Active Skills Reference Guide

This report lists the **14 core agent skills** retained in the `SS_Pharmacy` workspace for premium visual design, layout auditing, and accessibility.

| # | Skill Name | Purpose of Use / Description |
|---|---|---|
| 1 | `a11y-debugging` | Uses Chrome DevTools MCP for accessibility (a11y) auditing. Load when checking contrast, text size, and aria roles. |
| 2 | `accessibility` | Audit and improve web accessibility following WCAG 2.2 guidelines. |
| 3 | `brainstorming` | Used before any creative design or feature implementation to align user requirements. |
| 4 | `coding-standards` | Baseline coding standards and readable component formatting conventions. |
| 5 | `e2e-testing-patterns` | Best practices for writing and executing Playwright layout and screenshot tests. |
| 6 | `enhance-prompt` | Refine visual styling prompts for high-fidelity editorial UI generation. |
| 7 | `frontend-design` | High-fidelity editorial layouts, rejecting template-style AI designs. |
| 8 | `high-end-visual-design` | Premium typography, shadows, card structures, and spacing rules. |
| 9 | `minimalist-ui` | Earthy, minimalist warm cream layouts and flat grid aesthetics. |
| 10 | `systematic-debugging` | Resolving linter warnings and browser layout compilation failures. |
| 11 | `ui-polish` | Check and polish visual elements, margins, and letter spacing. |
| 12 | `ui-ux-core` | Core UI principles, color parity, and spring motion constants. |
| 13 | `ui-ux-pro-max` | Comprehensive design system patterns and touch-target check rules. |
| 14 | `verification-before-completion` | Verify code compilation, linting, and layout test runs prior to PR creation. |
